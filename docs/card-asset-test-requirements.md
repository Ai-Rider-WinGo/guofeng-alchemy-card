# 卡牌素材生成测试需求 v0.2

## 背景

当前 Windows + ComfyUI 素材生成已经跑通了初步样张。本轮测试从“能生成图”推进到“能稳定进入卡牌产品链路”：同一套卡牌需要同时满足原画质量、卡面裁切、缩略图识别、卡牌文字叠加、Cocos 资源导入和后续批量复现。

本轮仍然围绕秦汉 / 楚汉 MVP 小池，不扩成全量生产。

## 测试目标

1. 验证 3:4 卡牌主图在人物卡、地点卡、事件卡中的稳定性。
2. 验证卡框、卡背、缩略图和详情页大图是否能共用同一套原画。
3. 验证卡牌文字内容可以由配置/UI 叠加，不依赖 AI 图片直接生成文字。
4. 验证 ComfyUI workflow、seed、模型、LoRA、参考图、输出路径可追溯。
5. 验证首批 12 张 MVP 卡能形成“抽卡 - 点亮卡册 - 合成 - 详情阅读”的素材闭环。
6. 验证按钮、底部导航、筛选标签等交互 UI 与卡牌原画属于同一套国风历史卡牌视觉系统。

## 测试批次

### Batch A：已跑通样张稳定性复测

目的：确认已经成功的生成流程可以复现，不只是偶然出图。

卡牌：

- `liubang_002` 刘邦
- `jixin_002` 纪信
- `xiangyu_002` 项羽
- `zhanghan_002` 章邯
- `xingyang_escape_004` 荥阳脱困
- `julu_battle_004` 巨鹿之战

要求：

- 每张至少生成 2 个候选版本。
- 每张保留 1 个推荐版本。
- 每张保存对应 workflow JSON。
- 记录 seed、模型、LoRA、参考图、ControlNet/IP-Adapter 使用情况。
- 对比上一轮样张，确认风格没有漂到修仙、玄幻、二次元或页游风。

### Batch B：补齐 MVP 闭环卡

目的：补齐现有 `config/cards.json` 中还没有进入首批样张的卡牌，让抽卡池和合成链路有完整美术覆盖。

卡牌：

- `xingyang_001` 荥阳
- `julu_001` 巨鹿
- `hongmenyan_004` 鸿门宴
- `gaixia_siege_004` 垓下之围
- `chuhan_conflict_005` 楚汉相争
- `han_dynasty_founding_005` 大汉开国

要求：

- 地点卡不要做成普通风景照，需要保留“历史发生地 + 战略意味”。
- 事件卡必须体现事件关系，不接受单人站桩肖像。
- 5 级阶段事件卡可以更史诗，但不能变成玄幻神话。
- 画面中不要出现现代建筑、现代武器、现代服饰。

### Batch C：卡背与卡框测试

目的：验证抽卡翻牌、未解锁卡册、卡牌详情页都能使用统一视觉系统。

素材：

- `card_back_qinhan_001`
- `frame_common_001`
- `frame_uncommon_001`
- `frame_rare_001`
- `frame_epic_001`
- `frame_legendary_001`

要求：

- 卡背比例 3:4。
- 卡背可用于抽卡翻牌和卡册未解锁剪影。
- 卡框与原画分离，不把文字烘焙进原画。
- 稀有度差异通过边框、纹样、局部光效表达，不靠大面积刺眼发光。
- 中央可以有抽象印记或“国风炼金卡牌”标识，但必须避免文字乱码。

### Batch D：Cocos 导入验证

目的：确认素材能在 Cocos Creator 3.8+ 中实际使用。

每张推荐样张需要导出：

- 主图：`1024 x 1365` 或 `768 x 1024`
- 卡面预览图：同主图比例
- 缩略图：`256 x 341`
- UI 小缩略图：`128 x 171`

测试页面：

- 首页周期卡池 Banner 主推卡展示
- 抽卡结果页翻牌展示
- 卡册网格缩略图
- 卡牌详情页大图

验收重点：

- 主体不被卡框遮挡。
- 缩略图仍能看出卡牌类别和核心主体。
- 暗色 UI 中不会糊成一团。
- 亮色边框不会压过人物和事件主体。
- 卡牌文字区域预留足够空间。

### Batch E：交互按钮与控件风格修正

目的：修正“卡牌原画国风，但按钮像普通网页/模板 UI”的问题。

组件：

- 主按钮：抽 10 次
- 次主按钮：抽 1 次
- 合成按钮：立即合成
- 奖励按钮：领取
- 次级按钮：概率说明
- 图标按钮：关闭
- 底部导航选中态：抽卡
- 底部导航未选中态：首页 / 合成 / 卡册 / 我的
- 朱砂红点
- 禁用态按钮

要求：

- 以 `docs/ui-interaction-style-guide.md` 为验收依据。
- 以 `assets-source/prompts/ui-button-style.md` 为生成 prompt 基线。
- 按钮不烘焙中文文字，优先输出可叠字的透明 PNG 或九宫格切片。
- 每个主按钮至少有默认、按下、禁用、可领取/高亮四种状态。
- 底部导航必须像卡牌游戏铭牌，不像网页导航。
- 不使用蓝紫渐变、Material 扁平风、SaaS 胶囊按钮、玻璃拟态。

## 通用负面约束

所有卡牌都必须避免：

- 修仙
- 洞府
- 炼丹炉
- 符箓法阵
- 玄幻神怪化历史人物
- 现代武器
- 现代建筑
- 现代服饰
- 二次元萌化
- Q 版化
- 页游大红大金
- 恶搞历史人物
- 血腥断肢
- AI 乱码文字
- 水印和 logo 伪影

## 验收标准

### 单张图通过标准

- 一眼能看出是中国历史卡牌。
- 人物、地点、事件类型明确。
- 主体适合放入 3:4 卡框。
- 可以裁出 256 和 128 缩略图。
- 没有明显手部、武器、脸部、服饰结构崩坏。
- 没有 AI 乱码文字、水印、现代元素。

### 一组图通过标准

- 刘邦、纪信、项羽、章邯身份差异明显。
- 荥阳、巨鹿不是普通风景，有历史地点感。
- 鸿门宴、荥阳脱困、巨鹿之战、垓下之围能看出事件关系。
- 楚汉相争、大汉开国有阶段事件的史诗感，但不玄幻。
- 同一批图风格统一，可以进入同一卡池。

### 交互 UI 通过标准

- 抽 10 次比抽 1 次更有主按钮分量。
- 主按钮、次级按钮、图标按钮、底部导航有明确层级。
- 放在暗色国风卡牌 UI 中不突兀。
- 有青铜、漆木、宣纸、竹简或金石质感，但不堆满装饰。
- 默认、按下、禁用、选中、可领取状态清楚。
- 没有现代网页按钮感，没有修仙法阵或炼丹炉元素。
