# Windows ComfyUI 卡牌样张需求

## 背景

当前前端方向回到《国风炼金卡牌》策划案本身：竖屏移动端国风历史卡牌收集游戏。

本 PR 的目标不是继续优化旧 Next.js 前端 demo，而是给 Windows 机器上的 ComfyUI 明确一批可执行的卡牌样张生产需求，用于验证卡牌主图风格、事件图构图、卡面比例和后续 Cocos 资源接入。

## 制作目标

请在 Windows + ComfyUI 环境下先制作一组 MVP 样张：

- 人物卡 4 张
- 事件卡 2 张
- 卡背 1 张
- 可选：统一卡框/套版草稿 1 张

样张应服务于后续 Cocos 竖屏卡牌界面，而不是做成独立海报或宽屏插画。

## 统一视觉方向

必须符合以下方向：

- 国风半写实卡牌插画
- 中国历史人物与历史事件主题
- 古籍纸纹、金石纹理、青铜边框、低饱和金色点缀
- 暗色背景中保留宣纸或卷轴质感
- 人物卡突出身份、服饰、姿态和时代感
- 事件卡突出历史场景、双方势力、冲突关系和叙事氛围
- 原画与卡框尽量分离，方便后续统一套版

明确避免：

- 不要修仙、洞府、炼丹炉、符箓法阵
- 不要玄幻神怪化历史人物
- 不要二次元萌化、Q 版化、页游大红大金
- 不要现代武器、现代建筑、现代服饰
- 不要血腥、低俗、恶搞历史人物

## 输出规格

- 主图比例：3:4
- 建议尺寸：1024 x 1365，或 768 x 1024
- 文件格式：PNG
- 命名规则：与 `config/cards.json` 的 `image` 字段对齐
- 输出目录建议：`assets/cards/samples/`
- 每张图保留 ComfyUI workflow JSON，便于复现和调参
- 每张图至少保留 2 个候选版本，最终挑 1 个作为推荐样张

## 卡背要求

卡背用于抽卡、翻牌和未解锁图鉴。

- 比例：3:4
- 关键词：青铜纹、云纹、竹简、楚汉主题、低饱和金色线条、暗色底
- 中央可放“国风炼金卡牌”或抽象印记
- 不要炼丹炉、法阵、修仙符箓

## Prompt 基线

人物卡 prompt 结构：

`角色名 + 秦汉之际 + 身份 + 服饰关键词 + 场景关键词 + 情绪/姿态 + 国风半写实卡牌插画 + 竖版构图 + 古籍纸纹 + 金石纹理 + cinematic lighting`

事件卡 prompt 结构：

`事件名 + 秦汉之际 + 核心场景 + 双方势力 + 冲突关系 + 历史氛围 + 国风史诗插画 + 竖版构图 + 古籍纸纹 + 金石纹理 + cinematic lighting`

Negative prompt 建议：

`modern clothes, modern weapon, sci-fi, fantasy monster, immortal cultivation, magic circle, alchemy furnace, cute chibi, anime moe, over saturated red gold, gore, parody, text artifact, watermark, logo`

## 验收标准

第一轮样张通过标准：

- 一眼能看出是中国历史卡牌，不是修仙、玄幻或泛 fantasy
- 人物卡之间有身份差异：刘邦、纪信、项羽、章邯不能只是换脸换衣
- 事件卡必须体现事件关系，不接受单纯人物站桩图
- 卡牌主图适合放入 3:4 卡框，主体不被裁切
- 色彩能适配暗色国风手游 UI，不依赖大面积刺眼发光
- 文件命名、尺寸、workflow JSON 可追溯

## 与后续 Cocos 的关系

通过的样张会进入 Cocos 资源验证：

- 抽卡页主推卡展示
- 抽卡结果页翻牌展示
- 卡册网格缩略图
- 卡牌详情页大图

暂时不要求一次性生成全量卡池。先把这 6 张样张的风格跑准，再扩展楚汉主题卡池。
