# 后续开发顺序

当前开发阶段：**Pre-Alpha / Early Alpha**。

本文件按当前代码实现状态更新：`web/`、`server/`、`admin-light/`、`config/` 是后续正式开发主线；`client/cocos-client/` 已归档为 Archived Prototype，不再投入 Cocos 正式客户端开发。

## Project Status (2026-06)

MVP 完成情况：

- `web/` 已具备玩家核心页面，并通过 `/api` 代理连接后端。
- `server/` 已具备 NestJS + TypeORM + JWT + Swagger 的基础服务端。
- `admin-light/` 已具备 Vue 3 + Ant Design 轻量管理后台。
- `config/` 已具备共享玩法配置。
- SQLite 开发数据库可用，PostgreSQL 是生产环境规划。
- `client/cocos-client/` 已归档为 Archived Prototype，仅保留历史原型参考价值。

已实现功能：

- 玩家登录/注册、资料读取。
- 抽卡、当日剩余次数、库存、图鉴、碎片查询。
- 通用合成、配方合成、碎片兑换、Lv12 合成。
- 签到和玩家日常信息。
- 后台卡牌、卡池、配置、合成规则、用户、素材、审计和仪表盘接口。
- 轻量后台页面和开发代理。
- Seed 导入卡牌、卡池、规则和配置。

当前风险：

- 生产数据库迁移体系未完成。
- 抖音登录、广告激励、支付订单、风控和数据埋点未完成。
- Redis 已出现在生产配置中，但尚未在服务端模块中实际接入。
- 经济系统和关键接口测试不足。
- 素材生产仍偏本地脚本化，缺少对象存储/CDN 发布闭环。
- 管理后台权限粒度、审计闭环、配置发布和回滚机制仍需加强。

下一阶段开发重点：

- 稳定 Web/H5 玩家主链路。
- 完成抖音小游戏 H5 适配。
- 建立正式数据库 migration。
- 完成平台能力接入。
- 建立生产素材和配置发布流程。
- 补齐测试、CI、部署、监控和安全。

## 1. 玩家主链路稳定

- 端到端验证登录、抽卡、库存、图鉴、碎片、合成、签到、任务。
- 将游客态 localStorage 兜底和登录态服务端权威逻辑边界写清楚。
- 校验抽卡失败、合成失败、重复提交、token 过期和网络异常时的 UI 状态。
- 补齐任务系统的服务端进度、领取和奖励记录。
- 补齐周期收集奖励的领取接口和前端状态。

## 2. 数据库与后端生产化

- 移除生产环境对 TypeORM `synchronize: true` 的依赖。
- 建立 PostgreSQL migrations、rollback、seed 和环境隔离流程。
- 明确 SQLite 仅用于本地开发和快速验证。
- 补齐数据库索引、唯一约束和关键查询性能检查。
- 建立数据库备份、恢复、权限和审计策略。

## 3. 核心经济系统测试

- 抽卡概率和保底逻辑测试。
- 每日限制和重复请求测试。
- 卡牌消耗、碎片发放、库存变化测试。
- 合成成功、失败、失败回滚和无目标卡测试。
- 签到奖励、周期奖励和任务奖励测试。
- Seed 数据一致性测试。

## 4. 平台能力接入

- 抖音平台登录和用户身份绑定。
- 激励视频广告发奖校验。
- 支付订单创建、回调、验签和补单。
- 平台风控、异常行为识别和频率限制。
- 平台发布包、小游戏权限和审核材料准备。

## 5. 管理后台增强

- 管理员角色和权限粒度。
- 配置发布、预览、灰度和回滚。
- 卡池排期、限时 UP、补偿发放和公告管理。
- 审计日志闭环：谁在什么时间改了什么。
- 仪表盘接入真实运营指标。
- 素材任务状态和图片 URL 回填流程。

## 6. 素材生产与发布

- 固化卡牌图像命名、压缩、尺寸和质量标准。
- 将 ComfyUI / Colab / Python 脚本产物接入审核清单。
- 接入对象存储和 CDN。
- 在卡牌数据中维护 `image_url` 和 `thumbnail_url`。
- 建立素材版本管理和替换策略。

## 7. H5 / 抖音小游戏适配

- 保持 `web/` 为唯一正式玩家客户端入口。
- 按移动竖屏优先检查首页、抽卡、合成、卡册、签到、任务和个人信息页。
- 建立抖音小游戏 H5 容器适配层：登录、分享、广告激励、支付、埋点和审核限制。
- 梳理服务端权威边界：抽卡、库存、碎片、合成、签到、任务和奖励领取。
- 准备 H5 小游戏真机调试、平台审核材料和发布包。
- 保持 `client/cocos-client/` 为 Archived Prototype，不再接入正式联调或新增功能。

## 8. 工程化与部署

- 为 `web`、`server`、`admin-light` 建立 CI 构建检查。
- 增加 `.env.example`，规范本地、测试、生产环境变量。
- 完成 Docker 或云部署说明。
- 接入日志、监控、告警和错误追踪。
- 建立限流、CORS、上传大小、安全 header 和管理员操作保护。
- 形成上线 checklist 和回滚流程。
