# 后续开发顺序

## 1. 配置结构补齐

- 扩展 `draw_pools.json` 为常驻基础池、周期朝代池、限时高级池三层结构。
- 新增 `duplicate_conversion_rules.json`。
- 新增 `weekly_collection_rewards.json`。
- 新增 `dynasty_tags.json`。
- 将 1-2 级低阶卡也按当前周期主题投放。

## 2. Cocos 客户端页面

- 首页：当前周期卡池 Banner、卡册进度、今日任务。
- 抽卡页：主题、剩余时间、主推卡牌、单抽、十连、概率说明。
- 抽卡结果页：卷轴、水墨、卡背飞出、卡牌翻转、稀有度光效。
- 合成页：碎片合成、重复卡升星、低阶卡熔合。
- 卡册页：朝代筛选、类型筛选、已收集/未收集、朝代奖励。
- 卡牌详情页：历史简介、来源、升星与合成入口。

## 3. 玩法逻辑

- 抽卡时识别新卡与重复卡。
- 重复 1 级卡转同朝代碎片。
- 重复 2 级卡转升星材料或同朝代碎片。
- 重复 3 级及以上卡优先用于升星，也可分解为高级碎片。
- 每个周期卡池独立计算收集进度与奖励领取状态。
