# 《国风炼金卡牌》MVP 范围

当前开发阶段：**Pre-Alpha / Early Alpha**。

MVP 第一版验证“国风历史卡牌收集”闭环，不做复杂战斗和重世界观演出。

核心闭环：

```text
进入游戏
  -> 查看本周朝代主题卡池
  -> 免费抽卡 / 普通抽卡 / 高级抽卡
  -> 获得人物卡、事件卡、碎片或低阶卡
  -> 新卡点亮卡册
  -> 重复卡转化为碎片或升星材料
  -> 碎片足够后合成指定卡
  -> 提升本周主题收集进度
  -> 领取收集奖励
  -> 下周进入新朝代卡池
```

## 当前实际架构

```text
web/                 Web/H5 唯一正式客户端（Next.js 16 + React 19）
server/              NestJS + TypeORM 后端 API
admin-light/         Vue 3 + Ant Design 轻量管理后台
client/cocos-client/ Archived Prototype，历史 Cocos 原型
config/              共享游戏配置
SQLite               开发数据库
PostgreSQL           生产环境数据库规划
```

API 文档地址：

```text
http://localhost:3002/api/docs
```

## H5 / 抖音小游戏适配

MVP 正式客户端路线为 **Web/H5 单客户端**。抖音小游戏基于 H5 适配方案推进，不再继续投入 Cocos Creator 正式客户端开发。

MVP 适配范围：

- `web/` 页面按移动竖屏体验优先设计和验证。
- 登录、分享、广告激励、支付和埋点预留平台适配边界。
- 抽卡、库存、碎片、合成、签到和奖励领取保持服务端权威。
- `client/cocos-client/` 仅保留历史原型价值，不纳入 MVP 后续正式开发范围。

## MVP 必须覆盖

### 玩家前端

- 首页：当前周期卡池 Banner、卡册进度、今日任务、抽卡入口、合成入口。
- 抽卡页：当前卡池主题、单抽、十连、剩余次数、概率说明、卡池详情。
- 抽卡结果：新卡点亮、重复卡提示、碎片或升星材料提示、稀有度表现。
- 合成页：碎片合成、配方合成、通用升阶、可合成列表、碎片进度。
- 卡册页：总收集进度、朝代筛选、类型筛选、已收集/未收集状态。
- 卡牌详情页：主图、稀有度、朝代、类型、历史简介、来源、合成入口。
- 登录、签到、任务、个人信息等基础留存页面。

### 服务端 API

- 玩家注册、登录、资料读取。
- 服务端抽卡、库存、图鉴、碎片、每日抽卡次数。
- 合成规则校验、通用合成、配方合成、碎片兑换、Lv12 合成。
- 签到、玩家日常信息。
- 卡牌、卡池、配置、合成规则、用户、素材、审计、仪表盘等后台接口。
- Swagger API 文档。

### 管理后台

- 管理员登录。
- 卡牌管理。
- 卡池管理。
- 合成规则管理。
- 配置管理。
- 用户管理。
- 仪表盘。
- Prompt / 素材相关轻量管理入口。

### 数据与配置

- `config/cards.json` 和 `import_cards.json` 支撑卡牌导入。
- `config/draw_pools.json` 支撑卡池。
- `config/merge_rules.json` 支撑合成规则。
- `config/daily_limits.json` 支撑每日限制。
- `config/dynasty_tags.json` 支撑朝代标签。
- `config/duplicate_conversion_rules.json` 支撑重复卡转化。
- `config/weekly_collection_rewards.json` 支撑周期奖励。

## 周期卡池节奏

- 第 1 周：唐朝卡池
- 第 2 周：宋朝卡池
- 第 3 周：明朝卡池
- 第 4 周：三国卡池
- 第 5 周：秦汉卡池
- 第 6 周：春秋战国卡池

## 成功标准

- 用户能在 3 分钟内理解：抽卡、重复卡、碎片、升星、合成、卡册。
- 用户能在一周内看到清晰的朝代主题收集目标。
- 即使抽到 1-2 级卡，也能推进当前朝代图鉴。
- 重复卡不会让用户感觉浪费。
- 登录态下关键经济行为由服务端记录和校验。
