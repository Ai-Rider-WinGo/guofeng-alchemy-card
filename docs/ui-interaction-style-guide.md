# 国风卡牌交互 UI 风格规范

## 目标

本规范用于修正“卡牌原画国风，但交互按钮像普通模板”的割裂问题。后续首页、抽卡页、合成页、卡册页、任务页、我的页面中的按钮、Tab、弹窗、资源栏、红点、进度条都应遵循这里的交互语言。

核心方向：

- 国风历史卡牌手游
- 青铜、漆木、宣纸、竹简、金石纹理
- 游戏 UI，不是网页按钮，不是 SaaS 控件
- 可点击区域清晰，移动端触控友好
- 有质感但不堆满花纹

## 视觉关键词

### 应该像

- 青铜描边
- 暗红漆面
- 墨黑底
- 宣纸浅底
- 细金边
- 竹简分割线
- 金石纹理
- 低饱和金色高光
- 轻微浮雕
- 卡牌游戏 Tab

### 不应该像

- 现代蓝紫渐变按钮
- SaaS 胶囊按钮
- Material Design 扁平按钮
- 网页导航条
- 霓虹赛博风
- 过度圆角玻璃拟态
- 页游大红大金粗暴发光
- 修仙符箓、法阵、炼丹炉纹样

## 色彩系统

### 主色

- 墨黑：`#16120E`
- 深漆红：`#5A1F1A`
- 青铜金：`#B88A3B`
- 暗金高光：`#D8B463`
- 宣纸底：`#E9D8B0`
- 竹简棕：`#7A4E2C`

### 功能色

- 可领取：`#D8B463`
- 已选中：`#B88A3B`
- 危险/不足：`#A64232`
- 禁用：`#5A5145`
- 新卡提示：`#E8C56F`
- SSR 强调：`#F1D47A`

### 使用原则

- 页面底色保持暗色或纸纹，不用纯白。
- 主按钮可以用深漆红 + 青铜金描边。
- 次级按钮用墨黑或宣纸底 + 细金边。
- 高亮只用于主操作、可领取、稀有度，不整页泛光。

## 按钮体系

### 1. 主操作按钮

适用：

- 抽 1 次
- 抽 10 次
- 立即合成
- 一键合成
- 领取奖励

视觉：

- 深漆红或墨黑底
- 青铜金外描边
- 内侧一圈暗金细线
- 轻微浮雕阴影
- 中央文字用浅金或宣纸色
- 左右可有短横纹或云纹收尾

状态：

- 默认：底色深、金边清晰
- 按下：整体下沉 2px，亮边变暗
- 可领取：增加小范围暗金呼吸光
- 禁用：去高光，文字变灰，纹理降低
- 加载：中间出现小型金色转轮或“处理中”

文案示例：

- 抽 1 次
- 抽 10 次
- 立即合成
- 领取

### 2. 高价值按钮

适用：

- 高级抽卡
- 限时卡池抽取
- SSR 概率提升入口

视觉：

- 墨黑底 + 双层青铜金边
- 边角可用青铜角花
- 只允许局部金色粒子，不用大面积强光
- 按钮上方可挂小标签：限时 / 提升 / SSR

注意：

- 高价值不等于更亮更俗。
- 不要使用紫蓝霓虹、宝石按钮、现代充值按钮风。

### 3. 次级按钮

适用：

- 概率说明
- 卡池详情
- 返回
- 筛选
- 取消

视觉：

- 宣纸底或半透明墨黑底
- 1px 青铜细边
- 文字用竹简棕或浅金
- 不加明显发光

### 4. 图标按钮

适用：

- 返回
- 关闭
- 设置
- 分享
- 帮助
- 详情

视觉：

- 正方形或圆角方形
- 青铜边框
- 墨黑半透明底
- 图标线条用浅金
- 点击态有轻微压暗

图标风格：

- 线性图标 + 少量金石转角
- 不使用现代填充彩色 icon
- 不使用 emoji 作为正式 UI icon

### 5. 底部导航 Tab

Tab 项：

- 首页
- 抽卡
- 合成
- 卡册
- 我的

视觉：

- 底栏像一段暗色木牍或青铜条
- 每个 Tab 是独立的压印铭牌
- 选中态：青铜金描边 + 上方小金线
- 未选中：低亮度线框
- 红点：小朱砂圆点，不用现代纯红气泡

选中态不要做成网页导航文字变色。它应该像“卡牌游戏底部功能铭牌”。

## 页面关键按钮

### 首页

主按钮：

- 进入本周卡池
- 去合成

次级入口：

- 今日任务
- 查看卡册

要求：

- 当前周期卡池 Banner 上的按钮要和 Banner 融在一起，像挂在画卷或青铜牌上的操作入口。

### 抽卡页

主按钮：

- 抽 1 次
- 抽 10 次

按钮层级：

- 抽 10 次是主视觉按钮，尺寸更宽。
- 抽 1 次是次主按钮，宽度略小。
- 概率说明和卡池详情必须是次级按钮。

要求：

- 不要做成两个普通网页按钮并排。
- 抽 10 次按钮可用青铜双边框和暗金局部高光。

### 合成页

主按钮：

- 立即合成
- 一键合成

次级按钮：

- 筛选
- 查看材料
- 分解重复卡

要求：

- 合成按钮要表达“卡牌融合/碎片合成”，不是炼丹炉、不是法阵。
- 可使用碎片纹理、卡牌叠层、竹简卷轴元素。

### 卡册页

控件：

- 朝代筛选
- 类型筛选
- 稀有度筛选
- 未收集剪影
- 收集奖励按钮

要求：

- 筛选控件像小型竹简标签，不像网页 select。
- 可领取奖励按钮使用暗金呼吸光。

## 尺寸建议

以竖屏移动端 750 x 1334 设计稿为基准：

- 主按钮高度：88-104 px
- 次级按钮高度：64-76 px
- 图标按钮：56-72 px
- 底部导航高度：112-132 px
- 按钮圆角：8-14 px，避免胶囊形
- 描边：2-4 px
- 内描边：1-2 px
- 文字字号：主按钮 30-36 px，次级按钮 24-28 px

## 动效规则

### 点击反馈

- 下沉 2px
- 亮边压暗
- 内阴影加深
- 100-160ms 完成

### 可领取提示

- 暗金小范围呼吸
- 周期 1.4-1.8s
- 不使用整屏闪烁

### 抽卡按钮强调

- 抽 10 次按钮可以有轻微金粉流动
- 粒子只沿边缘流动，不遮挡文字

## 验收标准

一套按钮通过标准：

- 放在暗色国风卡牌 UI 中不突兀。
- 一眼能判断主次操作。
- 抽 10 次比抽 1 次更有分量。
- 禁用、按下、选中、可领取状态清楚。
- 没有现代网页按钮感。
- 没有修仙、炼丹炉、符箓、法阵元素。
- 文字区清晰，不依赖图片生成文字。
